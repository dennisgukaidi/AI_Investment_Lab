# AI 投研系统开发规范 (v4.5 定稿版)

## 1. 运行环境与依赖
- **核心连接**：必须使用 `ib_insync` 处理 TWS 异步连接，严禁阻塞主线程。
- **配置信息**：IP `127.0.0.1`, 端口 `7496`, clientId `10`。
- **降级处理**：若 TWS 权限不足或无订阅，必须自动回退至 `yfinance` 补全数据，禁止报错中断。

## 2. 脚本编写准则 (Code Standards)
- **增量更新**：所有数据采集脚本（CSV/JSON）必须先检查本地已有数据，仅下载缺失部分，严禁暴力覆盖。
- **动静分离**：
  - `data/raw/`：存放原始量价 CSV。
  - `data/news/`：存放新闻摘要 JSON。
  - `scripts/`：存放所有逻辑 Python 脚本。
- **代码健壮性**：必须包含基本的 `try-except` 逻辑，针对 yfinance 频率限制需增加 `time.sleep` 重试机制。

## 3. 分析算法规范 (Analysis Logic)
- **蒙特卡洛模拟**：必须基于历史 HV/IV 执行不少于 5000 次模拟，计算概率分布。
- **风控矩阵**：止损位必须基于 ATR（平均真实波幅）动态计算，而非固定百分比。
- **研报排版**：
  - 必须使用 Markdown 引用块（`>`）进行核心结论强调。
  - 必须结合 `rules.md` 中的持仓成本计算回本概率。
- **数文分离**：禁止在 Python 脚本中硬编码 Markdown 排版。脚本仅负责输出 `metrics.json`。
- **Cline 职责**：读取 JSON 后，利用内置模型能力进行舆情摘要和趋势研判。
- **排版要求**：必须引用 JSON 中的 `prob_always_above` 和 `iv_percentile`，并配合 `> ` 引用块强调核心建议。
- **大盘基准**：流水线会下载 **SPY**；分析 JSON 含 `market_context`（SPY 趋势、相对超额、与标的 60 日收益相关）。撰写内参时应结合大盘与个股的 **regime 是否分化**。

## 4. 维护红线
- **禁止逻辑破坏**：更新 `rules.md` 表格时，严禁将观察清单（Watchlist）混入持仓列表。
- **备份机制**：在修改 `rules.md` 结构前，必须先生成 `.bak` 备份。
