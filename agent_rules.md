# Agent Rules for Script Updates (机器人更新脚本规则)

## 概述
本文件定义了当运行出现问题或需要修改/新增脚本时，机器人（AI助手）应遵循的规则和参数设置指南。所有脚本更新必须严格遵守 `cline_rules.md` 中的开发规范，并确保系统的稳定性和数据完整性。

## 触发条件
- 运行脚本时出现错误（Error 10089, 10358 等）
- 需要调整算法参数（蒙特卡洛模拟次数、时间窗口等）
- 添加新功能（新数据源、新分析指标）
- 优化性能或修复bug
- 扩展批量处理能力

## 核心规则

### 1. 环境与依赖检查
- **前置检查**：修改前必须验证 TWS 连接（端口 7496）、Python 环境激活、依赖包完整性
- **降级策略**：优先使用 TWS API，失败时自动回退至 yfinance，不允许中断流程
- **异步处理**：所有 TWS 操作必须使用 `ib_insync` 的异步机制，避免阻塞

### 2. 数据处理规范
- **增量更新**：禁止暴力覆盖现有数据，必须检查本地文件并仅下载缺失部分
- **文件命名**：严格遵循 `cline_rules.md` 的动静分离原则
  - 原始数据：`data/raw/{ticker}_180d.csv`
  - 新闻数据：`data/news/{ticker}_news.json`
  - 分析结果：`data/analysis/{ticker}_metrics.json`
- **数据验证**：下载后必须验证数据完整性（行数、列结构、时间范围）

### 3. 脚本修改参数指南

#### portfolio_pipeline.py 参数调整
- **OHLCV_CALENDAR_DAYS**：默认 500 日历日，支持 MA200 计算
- **BENCHMARK_SYMBOL**：固定为 "SPY"（标普500 ETF）
- **IV 计算窗口**：默认 30 日滚动 HV
- **重试机制**：TWS 连接失败时，增加 `time.sleep(5)` 重试 3 次

#### news_collector.py 参数调整
- **MAX_ITEMS**：默认 50 条新闻
- **MAX_DAYS**：默认 30 天保留期
- **INITIAL_BACKFILL_DAYS**：首次运行保留 30 天
- **去重逻辑**：基于标题和发布时间戳

#### analyze_report.py 参数调整
- **IV_PERCENTILE_ROWS**：默认 180 日 IV 分位计算
- **MC_SHORT_DAYS**：蒙特卡洛模拟短期窗口，默认 10 日
- **BOOTSTRAP_HORIZONS**：引导分析时间跨度，默认 (10, 20, 60) 日
- **BOOTSTRAP_SIMS**：模拟次数，默认 5000 次
- **BOOTSTRAP_RETURN_POOL**：历史收益池，默认 252 个交易日
- **风控倍数**：止损/止盈基于 ATR，默认 1.5x/2x

#### download_watchlist_data.py 参数调整
- **测试模式**：默认仅下载第一个股票，批量时修改为遍历全列表
- **DURATION**：默认 500 日历日
- **CLIENT_ID**：默认 10，可根据需要调整

### 4. 新脚本开发指南
- **功能定位**：明确脚本职责，避免功能重叠
- **输入输出**：定义清晰的输入文件和输出格式
- **错误处理**：包含完整的 try-except 块
- **日志记录**：添加适当的 print 语句或 logging
- **兼容性**：确保与现有流水线无缝集成

### 5. 性能优化规则
- **向量化计算**：优先使用 pandas/numpy，避免 Python 循环
- **内存管理**：大文件处理时使用分块读取
- **API 限流**：yfinance 请求间增加 sleep，避免频率限制
- **并发控制**：TWS 连接限制，避免同时多个客户端

### 6. 测试与验证
- **单元测试**：修改后运行现有数据验证输出正确性
- **边界测试**：测试极端情况（无数据、网络断开等）
- **回退机制**：确保修改失败时不破坏现有功能
- **文档更新**：修改脚本后同步更新本文件和 README.md

### 7. 常见问题解决方案

#### TWS 连接问题
- 检查端口 7496 是否开放
- 验证 IB Gateway/TWS 已登录
- 确认客户端 ID 未冲突
- 回退至 yfinance 数据源

#### 数据订阅错误 (Error 10089/10358)
- 联系 IB 客服开通相应市场数据订阅
- 临时使用 yfinance 补全基础数据
- 在日志中记录缺失字段

#### 内存不足
- 减少 BOOTSTRAP_SIMS 参数
- 使用数据分块处理
- 优化 pandas 数据类型

#### 网络超时
- 增加超时参数和重试机制
- 实现断点续传功能
- 添加代理支持

### 8. 更新日志
- **v4.5**：初始版本，建立基础规则框架
- 记录每次修改的日期、修改内容和测试结果

## 紧急处理
如果系统完全无法运行，优先恢复基础功能：
1. 验证 Python 环境和依赖
2. 检查 TWS 连接
3. 运行 download_watchlist_data.py 获取基础数据
4. 逐步恢复完整流水线

所有修改必须经过测试验证，确保不破坏现有自动化工作流。
